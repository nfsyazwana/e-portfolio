Project 1 – EcoEarn (MVP)

Problem Statement

Low recycling participation due to lack of waste segregation guidance and financial incentives. Households are often confused about proper disposal methods, and existing recycling programs offer no tangible rewards for effort.

Working Features

  • User authentication via SharedPreferences
  • Item category selection (Plastic, Glass, Paper, Metal, E-waste)
  • Points accumulation and reward catalog
  • Form submission with photo proof upload
  • Cash redemption to e-wallet
GitHub Repository YouTube VSR

Lessons Learned

Developed a complete user flow from registration to reward redemption. Strengthened understanding of Android lifecycle management, data persistence, and user-centered design. Learned to debug real-world integration issues and implement business logic for point-based systems.

Project 2 – EcoEarn Enhanced

Problem Statement

Building upon the MVP, Project 2 required extending EcoEarn into a fully-featured platform with real-time synchronization, sensor-based gamification, and external API integration to provide users with nearby recycling center information.

GitHub Repository YouTube VSR

Lessons Learned

Implementing a production-grade Android application required mastering several advanced concepts. I integrated Firebase Authentication and Firestore for secure cloud synchronization, ensuring user data persistence across devices. Room Database was used for offline-first capabilities, allowing seamless operation without network connectivity. Retrofit enabled REST API consumption to fetch live recycling center data, while Google Maps SDK provided visual location context. Sensor integration (step counter) introduced gamification elements that increased user engagement. The MVVM architecture pattern combined with Coroutines and Hilt dependency injection resulted in a maintainable, testable codebase. Key challenges included managing state across asynchronous operations, handling API rate limits, and optimizing database queries for performance. This project solidified my ability to deliver complex, multi-feature Android applications independently.

Reflection & Learning Journey

The progression from Lab 1 to Project 2 represents a comprehensive learning arc. Initial labs established fundamentals: UI design, event handling, Material guidelines, navigation patterns, and local persistence. Project 1 challenged me to synthesize these skills into a functional application addressing a genuine user problem — confusion about recycling and lack of incentives. Project 2 elevated the complexity significantly: cloud synchronization, API integration, sensor-based features, and production-ready architecture (MVVM, Coroutines, Hilt).


Significant challenges included debugging Room migration conflicts, managing asynchronous data streams, and ensuring offline functionality worked seamlessly with cloud sync. Each obstacle reinforced the importance of systematic debugging and architectural planning. SDG 12 remained the guiding framework — every feature directly supports responsible consumption by making recycling easier and financially rewarding. This portfolio demonstrates technical competency and a commitment to using mobile technology for environmental impact.
